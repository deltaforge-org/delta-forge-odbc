## What's Changed

Alignment of the DeltaForge ODBC driver to the 1.2.0 platform release.

- The driver works unchanged against the new compute gateway and the cloud autoscaler pool. It already dials a compute node directly over the deployment's public endpoint and mirrors the control plane's scheme (http or https) automatically, so a query that lands on a freshly warmed cloud slot behaves exactly like one against a fixed node, with no driver configuration change.
- Compute-node routing and client usage attribution are as introduced in 1.0.8: every query carries the `x-deltaforge-node` selector (direct-dial when a node is pinned, server-side load-balancing when it is not), and requests are tagged as ODBC traffic so DeltaForge's per-table and fleet usage dashboards record when the lake was reached through ODBC (Power BI, Excel, Tableau, DBeaver, and other ODBC tools).
- Versioned to the 1.2.0 platform release.

Result handling, type mapping, and `SQLGetData` behaviour are unchanged from 1.0.8; this is a version-alignment release with no change on the wire.
